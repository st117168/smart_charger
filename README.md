Program which manage windows/linux power charging via wifi-support smart plug Tuya. 

# Smart Charger

Умное управление зарядкой ноутбука через смарт-розетку Tuya. Автоматически включает/выключает розетку в зависимости от уровня заряда батареи.

## Требования

- **Windows**: Python 3.9+
- **Linux**: Python 3.9+, systemd (для автозапуска)

---

## Установка

### Windows

1. Запустите `windows/install.bat` от имени администратора
   - Создаст виртуальное окружение `.venv`
   - Установит зависимости (`tinytuya`, `psutil`, `pywin32`, `wmi`)
   - Создаст папку `data` и шаблон `config.json`
   - Настроит задачу в планировщике Windows для автозапуска

2. Отредактируйте `data/config.json` — укажите данные вашего Tuya устройства

3. Перезагрузите компьютер или выполните ручной запуск для проверки

### Linux

1. Запустите `linux/install.sh`
   - Создаст виртуальное окружение `.venv`
   - Установит зависимости (`tinytuya`, `psutil`)
   - Создаст папку `data` и шаблон `config.json`
   - Настроит systemd-сервис для автозапуска при входе в систему

2. Отредактируйте `data/config.json` — укажите данные вашего Tuya устройства

3. Перезагрузите компьютер или выполните ручной запуск для проверки

## Конфигурация

Отредактируйте `data/config.json`:

```json
{  
    "tuya": {  
        "device_id": "ID_УСТРОЙСТВА",  
        "local_key": "ЛОКАЛЬНЫЙ_КЛЮЧ",  
        "ip_address": "",  
        "version": 3.5  
    },  
    "battery": {  
        "min_level": 30,  
        "max_level": 80,  
        "check_interval": 60  
    }  
}  
```

## Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `tuya.device_id` | string | ID устройства Tuya (обязательно) |
| `tuya.local_key` | string | Локальный ключ устройства (обязательно) |
| `tuya.ip_address` | string | IP адрес (оставьте пустым для автоопределения) |
| `tuya.version` | float | Версия протокола (3.3 или 3.5) (у меня рабтает только с 3.5) |
| `battery.min_level` | int | Минимальный заряд для включения розетки (%) |
| `battery.max_level` | int | Максимальный заряд для выключения розетки (%) |
| `battery.check_interval` | int | Интервал проверки (секунды) |

## Просмотр логов

### Linux (Bash)
* **Мониторинг сервиса в реальном времени:**
    ```bash
    journalctl --user -u smart-charger.service -f
    ```
* **Просмотр последних 50 строк:**
    ```bash
    journalctl --user -u smart-charger.service -n 50
    ```
* **Чтение файла лога напрямую:**
    ```bash
    tail -f data/smart-charger.log
    ```

### Windows (Text/CMD)
* **Файл лога:** Находится по пути: `data\smart-charger.log`
* **Просмотр задачи планировщика:**
    ```cmd
    schtasks /query /tn "SmartChargerTask" /v
    ```

## Устранение неполадок

### 1. Программа не запускается
* **Конфигурация:** Проверьте наличие и корректность файла `data/config.json`.
    * Поля `device_id` и `local_key` должны быть заполнены.
    * Проверьте формат JSON на валидность синтаксиса.
* **Сеть:** Убедитесь, что устройство Tuya доступно в сети.
    * Розетка должна быть подключена к Wi-Fi.
    * IP-адрес устройства должен пинговаться.
* **Логи:** Проверьте файл `data/smart-charger.log` на наличие специфических ошибок.

### 2. Windows: Задача планировщика не создается
* Запустите файл `windows/install.bat` **от имени администратора**.
* Проверьте наличие задачи через командную строку:
    ```cmd
    schtasks /query /tn "SmartChargerTask"
    ```

### 3. Linux: Сервис не запускается
* **Проверить статус:**
    ```bash
    systemctl --user status smart-charger.service
    ```
* **Просмотреть ошибки:**
    ```bash
    journalctl --user -u smart-charger.service -n 20
    ```
* **Проверить права:** Убедитесь, что интерпретатор доступен:
    ```bash
    ls -la .venv/bin/python
    ```

### 4. Частые переключения розетки
* Если программа включает/выключает розетку слишком часто, увеличьте параметр `check_interval` в `config.json`.
* *Рекомендуемое значение:* **60-120** секунд.

## Особенности работы по ОС

### Windows
* **Автозапуск:** Реализован через Планировщик задач (запуск при старте системы).
* **Интеграция:** Отслеживает события сна и пробуждения системы через WMI.

### Linux
* **Автозапуск:** Реализован через `systemd` (user unit), запускается при входе пользователя.
* **Фон:** Работает как полноценный демон в фоновом режиме.
* **Логирование:** Вывод доступен одновременно через `journalctl` и в файл `data/smart-charger.log`.
